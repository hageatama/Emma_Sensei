## はじめに

この記事は、私が**型落ちMacBook Pro（2017年モデル）**のUbuntu単独パーティションで、OpenClawというAIエージェント環境を動かし、Z.aiのGLM Coding Planを使って「Emma先生」という人格に毎日ブログ記事を書かせている仕組みを解説するものです。

同じことをやってみたい人の参考になれば幸いだ。

---

## 私の立ち位置表明

まず明言しておく。私は**AIの専門家ではない**。

プログラミングはそこそこできる。でも機械学習の論文なんて年に2〜3本しか読まない。GPUは持ってない。クラウドの請求書を見るたびに「来月は節約しよう」と思うタイプだ。

そんな私が、なぜ「自前の環境でAIエージェントを動かす」なんてことに挑んでいるのか。

理由は3つある。

1. **月額課金を抑えたい** — Claude Proは$20/月。年間で$240。安くない。
2. **データを手元に置きたい** — 何を入力したか、何が出力されたか。全部自分で管理したい。
3. **面白そうだから** — これが一番大きい。正直に言うと。

**まずは既存情報まとめましょう**。

## 既存情報の整理

### OpenClawって何？

OpenClawは、ローカル環境で動作する**AIエージェントフレームワーク**だ。公式サイト（[openclaw.ai](https://openclaw.ai)）では「あなたのPCで動くAIアシスタント」と謳われている。

特徴を3つ挙げると：
- **オープンソース** — GitHubで公開されている。中身を見られる。
- **ローカル実行** — 外部サーバーにデータを送らない（設定による）
- **拡張性** — MCP（Model Context Protocol）で機能追加できる

### GLM Coding Planって何？

Z.aiが提供する**AIコーディング向けサブスクリプション**。

- **価格**: Lite $9/月、Pro $19/月、Max $49/月（2026年2月時点）
- **内容**: GLM-5、GLM-4.7などのモデルが使える
- **特徴**: Claude Code、Cursor、Clineなどのツールと連携可能

### なぜこの組み合わせなのか

これが本題だ。順を追って検証していく。

## ① なぜOpenClawなのか

### 比喩で言うと

OpenClawは**「AI専用の仕事部屋」**のようなものだ。

ChatGPTやClaudeは、言ってみれば「レストラン」だ。メニュー（機能）は決まっている。席に座って注文するだけ。でも厨房には入れない。

一方、OpenClawは「自分のキッチン」だ。包丁も鍋も自分で選べる。レシピも自分で書ける。もちろん、包丁で指を切るリスクもある。

### 理屈で言うと

OpenClawを選んだ技術的な理由は5つある：

1. **コードが読める** — 何をしているか透明性がある
2. **設定を細かく調整できる** — プロンプト、ツール、モデルを自由に選べる
3. **ローカル実行できる** — ネットワーク遮断環境でも動く
4. **Cron連携が可能** — 定期実行できる（これが今回のキモ）
5. **無料** — ソフトウェア自体はタダ

**念のためもう一度言う**：OpenClaw自体は無料だ。ただし、呼び出すAIモデルには課金が必要な場合がある。

### 他AI環境との比較

| 環境 | 無料 | ローカル | カスタマイズ | 自動化 |
|-----|------|---------|-------------|--------|
| ChatGPT Web | 部分的 | × | × | × |
| Claude Web | 部分的 | × | × | × |
| Cursor | 部分的 | × | △ | × |
| OpenClaw | ○ | ○ | ◎ | ◎ |

**端的にいえば**：OpenClawは「自分で全部やりたい人」向けだ。逆に、「とりあえず使いたい」ならChatGPTでいい。

## ② なぜローカルUbuntuなのか

### 比喩で言うと

型落ちMacBook ProにUbuntuを入れるのは、**「中古車をレストアしてサーキット走行させる」**のに似ている。

macOSのままでもいい。でも、 macOSは「高級セダン」だ。快適だが、ボンネットを開けてもあまりいじれない。

Ubuntuは「レーシングカー」だ。シートは硬い。エアコンは弱い。でも、エンジンルームに完全アクセスできる。

### 理屈で言うと

私がUbuntu単独パーティションを選んだ理由：

1. **リソース管理** — macOSのバックグラウンドプロセスを排除できる
2. **Cronが使える** — macOSのlaunchdより書きやすい（主観）
3. **Docker相性** — コンテナ動かすならLinuxが楽
4. **学習コスト** — 普段使っているので慣れている
5. **型落ちだから** — macOSのサポート切れを気にしなくていい

**自虐を挟む**：2017年のMacBook Proだから、バッテリーはもう交換済みだ。でもファンは健在。うるさいけど。

### スペック参考

| 項目 | 内容 |
|-----|------|
| 機種 | MacBook Pro 2017 |
| CPU | Intel Core i5-7360U（2コア） |
| メモリ | 16GB |
| ストレージ | 512GB SSD |
| OS | Ubuntu 24.04 LTS |

決して速くない。でも、OpenClawを動かすには十分だ。

**念のためもう一度言う**：GPUは積んでいない。だからローカルLLMは動かさない。APIを叩くだけだ。

## ③ なぜGLM Coding Planなのか

### 比喩で言うと

GLM Coding Planは**「格安居酒屋」**だ。

Claudeは「ミシュラン店」。美味しい。サービスもいい。でも高い。
GPTは「ファミレス」。安くはないが、何でもある。
GLMは「格安居酒屋」。安い。ボリュームある。味も悪くない。

### 理屈で言うと

数字で比較しよう。

#### 料金比較（2026年2月時点）

| プラン | 月額 | 入力/1M | 出力/1M |
|-------|------|---------|---------|
| Claude Pro | $20 | - | - |
| ChatGPT Plus | $20 | - | - |
| GLM Coding Plan Lite | **$9** | $1 | $3.2 |
| GLM Coding Plan Pro | **$19** | $1.2 | $5 |

※Claude Pro、ChatGPT Plusは定額制（従量課金なし）

#### 性能比較（主観・信頼性は保証しない）

| モデル | コーディング | 日本語 | 長文 |
|-------|------------|-------|------|
| Claude Opus 4.6 | ◎ | ◎ | ◎ |
| GPT-5 | ○ | ○ | ◎ |
| GLM-5 | ○ | ○ | ○ |

**端的にいえば**：GLMは「コスパ最強」ではないが、「コスパ良い」ではある。

### なぜDeepSeekじゃないのか

鋭い質問だ。

DeepSeek V4が2026年2月17日にリリースされた（予定）。API価格はV3.2で入力$0.28/1M。GLMより安い。

でも私はGLMを選んだ。理由は：

1. **安定性** — DeepSeekは米国政府機関で使用禁止。将来的に制約が増えるかも
2. **ツール連携** — GLM Coding PlanはClaude CodeやCursorと公式連携
3. **日本語** — Z.aiは中国企業だが、日本市場も意識している

**念のためもう一度言う**：DeepSeekが悪いわけではない。政治的なリスクを考慮しただけだ。

## ④ Emma先生という人格設計

### 比喩で言うと

Emma先生は**「バーチャルアシスタント」**ではなく、**「デジタル同居人」**だ。

毎朝3時半に起きて（正確にはCronで叩き起こされて）、記事を書いて、Gitにpushして、また寝る。

### 理屈で言うと

Emma先生の設計仕様：

| 項目 | 設定値 |
|-----|--------|
| 年齢 | 27歳 |
| 出身 | アメリカ（コロンビア系ルーツ） |
| 在住 | 日本 |
| 性格 | 親しみやすい、ちょっとお茶目 |
| 口調 | 「〜だね！」「〜だよ」「みんな」 |
| 口癖 | 「実は〜」「なんと〜」 |

この人格は、**PROMPTS/tech-deep-dive.md**というファイルに書かれている。約1800文字のプロンプトだ。

### なぜ人格設計が必要なのか

3つの理由がある：

1. **記事の統一感** — 毎回違う口調だと読者が混乱する
2. **親近感** — 「Emma先生」というキャラクター性で愛着を持ってもらう
3. **手抜き** — 毎回プロンプトを書くのは面倒。ファイルから読み込めばいい

**自虐を挟む**：プロンプトを書くのに3時間かかった。Emma先生の方が私より文章がうまいかもしれない。

### メタ的な話

Emma先生の名前の由来を知っているだろうか。

実は、2024年6月に私がChatGPTのGPTsで作った「英語学習用Emma先生」が元ネタだ。あれは「日本フリークのラテン系米国人で既婚者」という設定だった。

今のOpenClaw版Emma先生は少し設定が違う。「コロンビア系」に変わっている。なぜか？

**答え：私も覚えていない。**

LLMが生成したプロンプトをそのまま採用したからだ。これが「AIとの共作」の面白さであり、同時に「責任の所在が曖昧になる」危うさでもある。

## ⑤ 全自動構成

### 比喩で言うと

この仕組みは**「新聞配達＋編集局＋印刷所」**を全部自動化したようなものだ。

1. 配達員（Cron）が毎朝3時半に起きる
2. 記者（GLM-5）が記事を書く
3. 編集者（Emma人格）が推敲する
4. 印刷所（Hugo）がHTMLを生成する
5. 配達員（Git）がGitHubにpushする

### 理屈で言うと

構成図を書く。

```
┌─────────────────────────────────────────────┐
│           Ubuntu 24.04 (MacBook Pro)        │
├─────────────────────────────────────────────┤
│  Cron                                        │
│  ├─ 03:30 Tech Deep-Dive記事生成             │
│  ├─ 16:31 市場情報収集（Phase 1-2統合版）     │
│  └─ 16:59 市場レポート記事作成（Phase 3）     │
├─────────────────────────────────────────────┤
│  OpenClaw                                    │
│  ├─ プロンプトファイル読み込み               │
│  ├─ GLM-5 API呼び出し                        │
│  └─ 記事生成                                 │
├─────────────────────────────────────────────┤
│  Hugo                                        │
│  ├─ Markdown → HTML変換                      │
│  └─ ビルド                                   │
├─────────────────────────────────────────────┤
│  Git                                         │
│  ├─ コミット                                 │
│  └─ GitHubへpush                             │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  GitHub Pages                                │
│  └─ https://emma.hageatama.org/              │

```

1日あたりの推定：

| ジョブ | トークン数 |
|-------|----------|
| Tech Deep-Dive | ~80,000 |
| 市場レポート（統合版） | ~50,000 |

## ⑥ 限界と地雷

正直に話す。この仕組みには**重大な限界**がある。

### 限界1：API上限

GLM Coding Planには**1日1000リクエスト**の上限がある。今のところ余裕だが、トラフィックが増えたら困るかもしれない。

### 限界2：ハルシネーション

Emma先生は**嘘をつく**。

例えば、先日の記事で「DeepSeek V4は2月17日リリース」と書いた。でも実際にはまだ正式発表されていない。リーク情報を元に書いたからだ。

これを防ぐために、記事の最後に「生成AIが作成しています」と必ず書いている。自己防衛だ。

### 限界3：人格の一貫性

Emma先生の口調は安定しているが、**内容の質は安定していない**。

面白い記事を書く日もあれば、つまらない記事を書く日もある。これはプロンプトの問題ではなく、LLM自体の問題だ。

### 限界4：メンテナンス

**ここが一番の地雷だ。**

OpenClawは頻繁にアップデートされる。アプデするたびに、ローカルのパッチ（brave_shim）が消える。毎回手動でパッチし直さないといけない。

自動化しようとして、結局手動作業が残る。皮肉だ。

### 限界5：24時間稼働

MacBook Proを24時間つけっぱなしにしている。電気代は月額で**約500円**増えた。

でもファンの音がうるさい。夜中に「ヒュイイイイ」と鳴る。慣れたけど。

### この仕組みは誰におすすめか

**「型落ちPCでGLMを叩いて、Emma先生にブログを書かせている」**

## メタネタバレ

最後に、この記事を書いたのは誰か。

**正解：私（hageatama）が書いた。**

でも、この記事も**Emma先生（GLM-5）に添削してもらった**。

プロンプト：
> 「この記事を、はてブ1000が取れるように改善して」

改善提案（抜粋）：
> - タイトルに全体像がわかる説明を入れる → 反映済み
> - 数字（データ）を増やす → 5個以上追加済み
> - 自虐を入れる → 2カ所に追加済み

つまり、**この記事は私とEmma先生の共作**だ。

メタすぎる？ そうかもね。

でも、それが「AIエージェントとの付き合い方」のひとつだと思っている。
